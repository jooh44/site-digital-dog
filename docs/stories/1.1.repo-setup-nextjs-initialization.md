### File Naming Conventions

[Source: architecture/coding-standards.md]

- Components: PascalCase (Button.tsx)
- Utilities: camelCase (analytics.ts)
- Pages: lowercase (page.tsx)
- Constants: UPPER_SNAKE_CASE

### TypeScript Configuration

[Source: architecture/coding-standards.md]

- Use strict mode (`"strict": true`)
- Prefer interfaces over types for objects
- Use enums for fixed values
- Avoid `any` - use `unknown` if necessary

### Git Commit Messages

[Source: architecture/coding-standards.md]

Follow conventional commits format:

- `feat:` for new features
- `fix:` for bug fixes
- `docs:` for documentation
- `chore:` for maintenance tasks

Example: `feat: Initialize Next.js 14 project with TypeScript`

### Next.js Configuration Notes

[Source: architecture/deployment.md]

- Configure `output: 'standalone'` in next.config.js for Docker deployment
- This is required for the Dockerfile multi-stage build process

### Environment Variables

[Source: architecture/environment-variables.md]

Create `.env.example` with template structure:

- Database URL placeholder
- Analytics IDs placeholders
- Email API key placeholder
- Security secrets placeholders

### Testing

[Source: architecture/testing-strategy.md]

**Testing Standards:**

- Test file location: `__tests__/` directory or co-located with `*.test.tsx`
- Testing framework: Jest + React Testing Library (to be configured in later story)
- For this story: No tests required (infrastructure setup)

**Note:** Testing setup will be implemented in a later story. This story focuses on project initialization only.

### Project Structure Notes

- The `docs/` folder contains BMad artifacts and may be gitignored based on team preference
- The `ssh_run.py` file MUST be gitignored (contains VPS credentials)
- All environment files (`.env.local`, `.env`) should be gitignored

### Technical Constraints

- Next.js version must be 14.2 or higher
- TypeScript version must be 5.3 or higher
- Node.js version must be 20 LTS
- Must use App Router (not Pages Router)
