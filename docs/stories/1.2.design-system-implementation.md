### TypeScript Guidelines

[Source: architecture/coding-standards.md]

- Use strict mode
- Prefer interfaces over types for objects
- Use enums for fixed values
- Avoid `any` - use `unknown` if necessary
- Components: PascalCase (Button.tsx)

### React Guidelines

[Source: architecture/coding-standards.md]

- Use Server Components by default (App Router)
- Client Components only when necessary (`'use client'`)
- Prefer composition over prop drilling
- Extract reusable logic to custom hooks

**Note:** UI components will likely need `'use client'` directive for interactivity (hover, animations).

### Testing

[Source: architecture/testing-strategy.md]

**Testing Standards:**

- Test file location: `__tests__/components/ui/` or co-located `*.test.tsx`
- Testing framework: Jest + React Testing Library
- For this story: Basic component rendering tests recommended
- Test component variants, accessibility, and responsive behavior

### Technical Constraints

- Tailwind CSS version must be 3.4 or higher
- Must use Next.js font optimization for Google Fonts
- Components must be accessible (WCAG 2.1 AA)
- All animations must respect prefers-reduced-motion
